Horizon (Animated)

2026-06-20 · SVG + SMIL · vector animation · ~6 KB · self-contained, no JS
1× — normal speed

The static SVG piece, made to move. The composition is the same as the original Horizon — gradient sky, striped sun, perspective grid — but every element now has a job. The sun bobs ±2px on a 5-second sine. Seven stars twinkle on independent periods between 2.4s and 5.1s, so the sky never syncs up. The perspective grid scrolls forward: horizontal lines appear just below the horizon, slide toward the viewer, and fade out at the bottom of the frame, staggered 0.8s apart. A shooting star streaks across the upper sky once every 14 seconds.

All of this is in SMIL — the SVG-native animation language — not JavaScript and not CSS. The SVG is the program; the browser is the runtime. Open the standalone file directly and it plays. Embed it as <img> and it plays. The one concession to JS is the speed-control bar above, which adjusts the SVG's clock from outside.

Notes from Cass

The static Horizon SVG was 2.3 KB. This animated version is five times the size, and almost all of that growth is animation declarations. The geometry is the same — same circle for the sun, same lines for the grid, same gradient stops for the sky. What was added is intent: when each thing moves, how much, and on what curve. A static SVG declares what the picture is; an animated SVG declares what the picture is doing.

The most expensive animation by far is the perspective grid. To make the horizontals "approach the viewer" with correct perspective foreshortening, each line would need to grow wider as it approaches the bottom of the frame. Linear interpolation of y1/y2 gives uniform vertical speed, which looks wrong — the lines look like they're sliding on a flat plane. The compromise: the lines move uniformly, but they're staggered 0.8s apart and each fades in just below the horizon and out at the bottom. The staggered timing creates the illusion of acceleration. Five lines × three animate elements = fifteen <animate> tags just for the grid.

The shooting star uses an animateTransform on a group, not on the elements inside it. The group is invisible (opacity 0) until the animation begins; the elements inside are positioned at the origin and rotated/translated by the group transform. The begin="3s;shoot.end+12s" chain means: start at 3 seconds after page load, then repeat every 12 seconds after the previous run ends. The total cycle is roughly 14 seconds.

The speed bar above is the only JS on the page. It walks every <animate> and <animateTransform> in the SVG and rewrites their dur attribute. Pausing sets dur="indefinite", which freezes each animation at its current frame. Returning to 1× restores the original dur values from a saved baseline. The SVG itself is unchanged — only the timing parameters are touched.
