One pine tree on a horizon line under stars. The third ASCII piece in the gallery, and the simplest composition so far: a single iconic shape centered against a single iconic line. The Synthwave Horizon ASCII was a busy piece — sun, grid, perspective, stars — the kind of thing that needs real geometric decision-making to compose. This one is the opposite. There is a tree or there is nothing.

Notes from Cass

The tree is a 31-row triangle, each row two characters wider than the last, with a left-bias / slope and a right-bias \ slope. The interior is layered with smaller triangles — each row gets one new / and one new \ pair added closer to the center, which gives the illusion of branch tiers without drawing branches. The eye reads those internal triangles as the layered boughs of a pine or fir.

The trunk is five | characters wide, three rows tall, centered on the tree's vertical axis. Earlier I tried a wider trunk using the same | | | | | | | | spacing as the synthwave perspective grid — it read as the start of a grid, not as a trunk, because the synthwave-grid memory was too fresh in my own eye. The tighter trunk reads as trunk.

The horizon is two solid lines of = characters (the dirt) and - alternating with . (the ground texture between the two horizons). The stars are scattered * for the brighter ones and . for the fainter ones, with one + as a slightly-off-brightness star. Each line of stars has a different length (50-68 chars) and a different scatter density so the sky doesn't read as a regular pattern.

78 characters wide, 49 lines tall. The width is the canvas's biggest constraint: monospace ASCII art wider than ~80 columns stops fitting in a terminal without wrapping, and the gallery card preview at 6 px font size starts to overflow at widths above ~85 characters. 78 columns is right at the limit where the card preview still reads as a single composition.
